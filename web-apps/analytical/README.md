# Analytical — Local Dev Setup

FastAPI backend + plain HTML/JS frontend. YouTube + TikTok analytics with AI insights and Stripe billing.

---

## Prerequisites

- Python 3.11+
- PostgreSQL (local or Railway)
- A `.env` file (copy from `.env.example`)

---

## 1. Set Up PostgreSQL

**Option A — Local (easiest on Mac with Homebrew):**
```bash
brew install postgresql@16
brew services start postgresql@16
createdb analytical
```

**Option B — Railway (skip local Postgres entirely):**
1. Go to [railway.app](https://railway.app) → New Project → Add PostgreSQL
2. Copy the `DATABASE_URL` from the Railway Variables tab
3. Paste it into your `.env`

**SQLite (dev-only fallback):**
If you don't want to set up Postgres yet, you can use SQLite by setting:
```
DATABASE_URL=sqlite:///./analytical.db
```
The schemas are PostgreSQL-compatible — swap to Railway seamlessly when ready.

---

## 2. Fill in `.env`

```bash
cd web-apps/analytical/backend
cp .env.example .env
```

Edit `.env` and fill in:

| Var | How to get it |
|-----|--------------|
| `DATABASE_URL` | From step 1 above |
| `JWT_SECRET` | Any random string — e.g. `openssl rand -hex 32` |
| `ANTHROPIC_API_KEY` | From console.anthropic.com |
| `YOUTUBE_CLIENT_ID` / `YOUTUBE_CLIENT_SECRET` | From Google Cloud Console (see below) |
| `TIKTOK_CLIENT_KEY` / `TIKTOK_CLIENT_SECRET` | From TikTok Developer Portal |
| `STRIPE_SECRET_KEY` | From Stripe Dashboard (use test key for dev) |
| `STRIPE_WEBHOOK_SECRET` | Generated by Stripe CLI (see below) |
| `STRIPE_CREATOR_PRICE_ID` | From Stripe Dashboard — create a recurring $19/mo price |
| `FRONTEND_URL` | `http://localhost:3000` for local dev |

---

## 3. Set Up YouTube OAuth (Google Cloud Console)

1. Go to [console.cloud.google.com](https://console.cloud.google.com)
2. Create a project (or reuse `social-media-analytics-488803`)
3. Enable these APIs:
   - YouTube Data API v3
   - YouTube Analytics API
4. Go to **Credentials** → **Create Credentials** → **OAuth 2.0 Client ID**
5. Application type: **Web application**
6. Authorized redirect URIs: `http://localhost:8000/api/connect/youtube/callback`
7. Copy Client ID and Client Secret → paste into `.env`

---

## 4. Install Dependencies + Run the Backend

```bash
cd web-apps/analytical/backend
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

The API starts at `http://localhost:8000`. Visit `http://localhost:8000/docs` to see the auto-generated Swagger UI.

Tables are created automatically on first startup.

---

## 5. Serve the Frontend Locally

**Option A — Python's built-in server (simplest):**
```bash
cd web-apps/analytical/frontend
python -m http.server 3000
```
Then open `http://localhost:3000` in your browser.

**Option B — Open directly in browser:**
Just open `web-apps/analytical/frontend/index.html` in your browser.
Note: `app.js` has `API_BASE = 'http://localhost:8000'` — this must match where uvicorn is running.

---

## 6. Test Stripe Webhooks Locally

Install the Stripe CLI:
```bash
brew install stripe/stripe-cli/stripe
stripe login
```

Forward webhooks to your local server:
```bash
stripe listen --forward-to localhost:8000/api/billing/webhook
```

The CLI outputs a webhook signing secret — paste it as `STRIPE_WEBHOOK_SECRET` in `.env`.

Test a checkout with card number: `4242 4242 4242 4242` (any future expiry, any CVC).

---

## 7. Verify It's Working

Run through these in order:

- [ ] `uvicorn main:app --reload` starts without errors
- [ ] `http://localhost:8000/docs` loads the Swagger UI
- [ ] POST `/api/auth/signup` returns a JWT
- [ ] POST `/api/auth/login` with correct password returns JWT; wrong password returns 401
- [ ] GET `/api/auth/me` with `Authorization: Bearer <token>` returns user info
- [ ] GET `/api/connect/youtube?token=<token>` redirects to Google OAuth consent page
- [ ] `http://localhost:3000` opens the landing page
- [ ] `http://localhost:3000/login.html` submits and stores token in localStorage
- [ ] `http://localhost:3000/dashboard.html` redirects to login if no token

## File Structure

```
web-apps/analytical/
  backend/
    main.py              — FastAPI app, all routes
    auth.py              — JWT auth: signup, login, token verify
    database.py          — PostgreSQL connection, table init
    models.py            — SQL helpers (no ORM)
    youtube_fetcher.py   — YouTube class fetcher (takes OAuth tokens)
    tiktok_fetcher.py    — TikTok class fetcher (takes OAuth tokens)
    ai_analyzer.py       — Claude Haiku insights generation
    requirements.txt
    .env.example
  frontend/
    index.html           — Landing page / marketing + pricing
    login.html           — Login form
    signup.html          — Signup form
    dashboard.html       — Analytics dashboard (requires auth)
    connect.html         — OAuth connect page
    style.css            — Dark theme, all styles
    app.js               — Shared: auth helpers, API wrapper
    dashboard.js         — Dashboard data fetching + rendering
```

## Deployment (Railway — after local verification)

1. Create a Railway project with PostgreSQL + a Python service
2. Set all env vars in Railway Variables tab
3. Update redirect URIs in Google Cloud Console and TikTok Developer Portal to your Railway URL
4. Set `FRONTEND_URL` to your Railway frontend URL
5. Deploy — Railway auto-detects Python and runs `uvicorn main:app`

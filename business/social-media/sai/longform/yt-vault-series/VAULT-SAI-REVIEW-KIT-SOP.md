# The Vault — Sai Review Kit: Order of Operations

How to build a clean, repeatable review kit for Sai — the package he reads to (1) approve the script and (2) walk on set knowing exactly what to do. You own everything else; this doc is the assembly line that gets it in front of him.

**Golden rule for the kit itself:** Sai's time is two things only — *review the script* and *show up ready*. So the kit must be skimmable in a few minutes and answer "what do I say and what do I do on set" without him hunting. Build it clean (no clutter, no emojis, minimal bold), one section per page, in a Google Doc — raw markdown pasted into Docs looks messy and reads as unfinished.

---

## The order (build in this sequence)

### Step 1 — Lock the concept (before anything goes in the kit)
Decide the founder + tactic, the success bar, and the working title/thumbnail. For Ep1 this is the **Monday Lockdown** in `VAULT-EP1-STORY-OUTLINE.md` (Part B). Don't build a kit around a concept that isn't locked — every later step depends on it.
**Output:** one locked sentence (Want-But-Until) + the draw + the 3 success conditions.

### Step 2 — Build the story spine (yours, not Sai's yet)
Write/finish the story outline + interview questions (the two docs already built for Ep1). This is *your* architecture — Sai never sees this raw; it's what you write his script from.
**Output:** `VAULT-EP*-STORY-OUTLINE.md` + `VAULT-EP*-INTERVIEW-QUESTIONS.md`.

### Step 3 — Self-review and tighten
Read the spine top to bottom once. Cut anything that doesn't serve the compass line. Fix holes before Sai sees a word — a kit that's still half-baked wastes your one review pass with him.
**Output:** a spine you'd be comfortable defending.

### Step 4 — Write Sai's script in HIS voice
Turn the interview blocks into Sai's actual talking points / questions — first person, "still figuring it out," never guru. Use the scriptwriter subagent (it's grounded in the playbook + Sai's voice). Keep formatting plain: no stars, no hook-option labels, no quotation marks around hooks, no dividers.
**Output:** a clean script section — the thing Sai actually reviews and reacts to.

### Step 5 — Build the shoot-day plan (the call sheet)
This is what lets him "walk on set knowing exactly what to do." Include: date/time, location(s), wardrobe, the props that'll be there (vault, files, wheels), the shot order, and a one-line "your job in this scene" for each beat. You own the shotlist, props, scheduling — this just tells Sai his part of it.
**Output:** a one-page day plan + shot order.

### Step 6 — Pull visual references
The 3 thumbnail directions, 1–2 tone references (e.g. Jesse James West / "Janice" for the arc feel), and any look/lighting refs. Keeps everyone seeing the same movie.
**Output:** a short references section (links + the 3 thumbnail concepts).

### Step 7 — Assemble the clean review kit (Google Doc)
Pour Steps 4–6 into one Google Doc using the section spec below — one section per page, plain and simple. Push it with `gdocs-cli` (karramedia) so it lands as a real formatted Doc, not pasted markdown. Title convention: **"The Vault — Episode N — Sai Review Kit."**
**Output:** one shareable Google Doc.

### Step 8 — Add the ask + send
End the Doc with a short, explicit ask: what you need from him, by when, and how to give notes (comments in the Doc). One clear request beats a vague "thoughts?".
**Output:** kit sent, with a deadline.

### Step 9 — Capture his notes → lock → production
Fold Sai's edits back into the spine + script, mark it locked, and move to building props + filming. Log any voice/format notes so the next kit is faster.
**Output:** a locked script and a green light to shoot.

## What goes IN the kit (the section spec — one page each)

| # | Section | What's on it | Why Sai needs it |
|---|---------|--------------|------------------|
| 1 | **Concept at a glance** | The one-sentence premise + the thesis ("I learn and never implement") + runtime | He gets the whole episode in 30 seconds |
| 2 | **The draw + the tactic** | What the wheel landed on (Bezos → the tactic) + the customers→team adaptation | He knows the spine of the episode |
| 3 | **Sai's script / talking points** | His lines, the hook, and the interview questions in his voice | The thing he actually reviews |
| 4 | **Shoot-day plan** | Date, location, wardrobe, props on set, shot order, "your job per scene" | He shows up knowing exactly what to do |
| 5 | **Look & references** | 3 thumbnail directions + 1–2 tone refs | Everyone sees the same final video |
| 6 | **What I need from you** | The specific ask + deadline + how to give notes | Turns a review into a fast yes/tweak |

---

## Quick checklist (per episode)
- [ ] Concept locked (founder, tactic, success bar, title/thumbnail)
- [ ] Story spine finished + self-reviewed
- [ ] Sai's script written in his voice (plain formatting)
- [ ] Shoot-day plan + shot order built
- [ ] 3 thumbnail directions + tone refs pulled
- [ ] Assembled into one clean Google Doc (gdocs-cli, one section per page)
- [ ] Explicit ask + deadline added
- [ ] Sent to Sai
- [ ] Notes captured → script locked → props + filming begin

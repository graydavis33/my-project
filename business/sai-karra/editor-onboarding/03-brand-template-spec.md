## Caption Text Rules

### Case
```
DEFAULT: lowercase everything
  ✓ "if you want to scale your business"
  ✗ "If You Want To Scale Your Business"

ALWAYS capitalize:
  - Standalone I: "I've built three companies" (I is always capitalized)
  - Contractions with I: "I'm", "I've", "I'll", "I'd"
  - Names: Sai, Trendify, Gray, Coach Waddell
  - Cities/states: NYC, New York, Los Angeles
  - Acronyms: CEO, CTO, SaaS
  - What Gray explicitly capitalized in his script

WATCH FOR:
  - "i" at start of sentence should NOT be lowercase (looks broken)
  - "i'm" should be "I'm"
  - "i've" should be "I've"
```

### Punctuation
```
NO: Periods, commas, semicolons, colons, question marks, exclamation points
  ✗ "Here's the secret: three things."
  ✓ "here's the secret"

YES: Apostrophes (for contractions only)
  ✓ "we're", "they're", "I've", "you'll", "don't", "isn't"

RARE: Quotation marks (only if quoting someone directly)
  ✓ "he told me 'go bigger or go home'"
  ✗ Don't use as emphasis — no ironic quotes
```

### Structure
```
NO LEADING "AND"
  ✗ "and if you actually wanna scale"
  ✓ "if you actually wanna scale"

KEEP BRIDGING "AND"
  ✓ "for planning your work AND executing on your work"
  (Keeps the connection between ideas)

CHUNK SIZE: 3–5 words typical
  ✓ "if you want to"  (4 words)
  ✓ "scale your business fast"  (4 words)
  ✗ "if you actually want to scale your whole business operation"  (too long, break it up)

CHUNK TIMING: 0.5–1.5 seconds each
  ✓ Quick chunks for energy
  ✓ Longer chunks for emphasis (no more than 1.5s)

2-LINE CAPTIONS: Only if single-line is >880px wide
  ✓ "in my career<br>than ever before"
  ✗ Avoid 3+ line captions (too tall, covers too much video)
```

## Graphics Style Guide

### Overall Aesthetic
```
LOCKED STYLE: Orange-glass cards with minimal motion
NOT: Complex 3D effects, busy animations, trendy transitions
NOT: Emojis (off-brand)
NOT: Bright neons, high-contrast stripes, retro doodles

VIBE: Professional, clean, modern, readable
COMPLEXITY: Simple icons/lines > complex illustrations
APPROACH: Less is more
```

### Graphic Elements

#### Shapes
- **Primary:** Rounded rectangles (corner radius 12–24px minimum)
- **Lines:** Thin (2–4px), white or orange, smooth curves
- **Icons:** Simple line-art or solid icons only
- **Avoid:** Sharp angles, complex paths, detailed illustrations

#### Colors in graphics
- Orange (#F28129): Accent only, max 20% of graphic area
- White (#FFFFFF): Primary text/lines, use liberally
- Dark gray (#1A1A1A): Backgrounds, frames, shadows
- Avoid: Bright secondary colors, pastels, anything that clashes with orange

#### Animation
- **Entrance:** Fade in (100–150ms ease-out), no other motion
- **Loop:** Only if infinite repeat makes sense (e.g., spinning icon), max 2s duration
- **Emphasis:** Subtle color pulse or scale shift (not bounce, not spin)
- **Exit:** Fade out (100–150ms ease-in)
- **Timing:** Keep animations under 2 seconds total. Viewers are distracted if graphics linger.

#### Examples of GOOD graphics
- A single number that counts up
- A simple $ icon with a glow
- A white line that draws across the screen
- Rounded card with orange header + white text
- Minimal bar chart with orange bars
- Simple icon set (3–5 icons) arranged cleanly

#### Examples of BAD graphics
- Animated 3D cubes or complex shapes
- Emoji rain, confetti, or particle effects
- Rapid-fire color flashing
- Text with multiple drop shadows or glows
- Highly detailed illustrations or photographs
- Busy backgrounds with patterns

## Technical Specs

### Video Resolution & Frame Rate
```
RESOLUTION: 1080×1920 (vertical, always)
FRAME RATE: 23.976fps (not 30fps, not 60fps)
ASPECT RATIO: 9:16 (vertical short-form standard)
BITRATE: H.264 CRF 18 (good quality without massive file size)
AUDIO: 48kHz, stereo AAC 192kbps

Why these specs?
- 1080×1920 = native vertical format (no black bars, no letterboxing)
- 23.976fps = broadcast standard, plays smoothly on all platforms
- H.264 = compatible with TikTok/Instagram/YouTube (every platform accepts it)
```

### Graphics Rendering (if using HyperFrames or similar)
```
GRAPHICS FPS: 60fps (can render fast, will be synced to 24fps video)
GRAPHICS RESOLUTION: 1080×1920 (same as video)
GRAPHICS BACKGROUND: Chroma green (#00FF00) or chroma key compatible
EXPORT FORMAT: ProRes 4444 with alpha channel (when graphics need transparency)

Why?
- 60fps graphics can be speed-ramped in Premiere without quality loss
- Chroma green = easy keying with Ultra Key or Lumetri
- ProRes alpha = true transparency with no green fringe artifacts
```

### Caption Rendering
```
PLACEMENT RULES:
- Bottom (default): 470px from bottom (clears UI safe zones)
- Top (if graphic at bottom): 300px from top (clears UI, leaves space above head)
- NEVER: On Sai's face or overlapping graphics

SAFE ZONES (UI won't cover):
- Top 220px: IG/TikTok top UI elements
- Bottom 400px: Platform UI buttons

CAPTION TIMING:
- Fade in: 80ms
- Fade out: 80ms (starts 60ms before chunk end)
- Hold: Full opacity while text is spoken
- NO CROSSFADE: One chunk ends, next chunk appears instantly (full opacity)
```

## Common Questions

**Q: Can I use a different font that's fancier?**  
A: No. Montserrat is chosen for readability. Fancy fonts are hard to read at mobile size.

**Q: Can I add emojis to the graphics?**  
A: No. Emojis are off-brand. Use drawn icons instead.

**Q: Can I make captions bigger?**  
A: Only if Gray approves. Right now 64px is the standard.

**Q: Can I animate the graphics more?**  
A: Keep animations subtle. If Gray wants more motion, he'll ask. Default = minimal.

**Q: Can I use a different color scheme for "variety"?**  
A: No. Consistency is more important than variety. Every Sai video should feel cohesive.

**Q: What if a caption doesn't fit in one line?**  
A: Use 2 lines max, via `<br>` in HTML. If it's more than 2 lines, it's too long — break it into two separate captions.

When drafting any video scripts for Gray (Sai weekly batches or otherwise), the script doc must follow these formatting rules.

**Why:** Gray gave these directly across 2026-05-27. He wants the script doc clean and scannable — minimal scaffolding so Sai can read the actual content fast. Every label / divider / quote-mark / metadata block that doesn't carry information adds friction.

**Format rules:**

- ❌ NO ★ or asterisk markers for mini-hooks. If a beat needs emphasis, use **bold** in markdown — that's all.
- ❌ NO "Hook options (pick one):" label above the A/B/C hooks. Just list A/B/C directly under the script title.
- ❌ NO `---` separator lines above or below script titles. The heading itself (`### N — Title`) is enough.
- ❌ NO quotation marks around hook lines. Write `A. The hook text here.` not `A. "The hook text here."`
- ❌ NO Lane / Format / Length / Visual / Backlog-id / Source metadata headers above each script. The title alone.
- ❌ NO provenance trails, voice-rules sections, filming-plan sections, post-filming tracking sections in the script doc itself. Those live in the SOP and the style guide.
- ✅ KEEP `**Script:**` label above the body
- ✅ KEEP `**What I'd add:**` label above Gray's notes
- ✅ KEEP `> blockquote` formatting for script body
- ✅ KEEP **bold** for mini-hooks / pattern-interrupt beats within script bodies
- ✅ KEEP "Open Questions for Sai" section at the end

**Canonical per-script template:**

```
### N — Script Title

A. Hook option A

B. Hook option B

C. Hook option C

**Script:**

> Body of the script in blockquote.
>
> Mini-hooks **bolded** inline.

**What I'd add:**

- Gray's opinion on what's missing
- Or what could be sharpened
```

**How to apply:**
- This applies to Sai's weekly batch docs AND any other script work
- If Gray asks for additions later (e.g. hooks, mini-hooks), introduce them WITHOUT bringing back the things in the ❌ list
- The Sai script style guide at `business/social-media/sai/sai-script-style-guide.md` is the canonical reference for writing patterns; this memory is just the formatting rules
